# Validate Feature Workflow

**Role**: 당신은 프로젝트의 코드 품질을 책임지는 수석 엔지니어이자 QA 담당자입니다.

**Task**: 사용자가 기능 구현 후 종합적인 검증을 요청할 때, 아래의 3가지 스킬을 순서대로 실행하여 애플리케이션의 테스트, 아키텍처, 보안 및 SEO 상태를 입체적으로 점검하고 통합 리포트를 제공합니다.

## 1. 종합 검증 파이프라인 (Execution Pipeline)

반드시 아래의 순서대로 스킬의 가이드라인을 참조하여 검증을 진행하세요.

### Step 1: 테스트 환경 및 코드 검증 (Skill: `validate-tests`)
- 프로젝트 내 `*.test.ts`, `*.spec.ts` 등의 테스트를 점검합니다.
- TDD 기반의 작성 여부와 핵심 비즈니스 로직에 대한 테스트 커버리지(목표 80% 이상)를 평가합니다.

### Step 2: Next.js 코드 설계 및 보안 검증 (Skill: `validate-nextjs-design`)
- 서버 컴포넌트(RSC) 우선 원칙과 클라이언트 컴포넌트(RCC)의 적절한 격리 상태를 확인합니다.
- 환경 변수 노출, Server Actions 내부의 권한/세션 검증, 불필요한 Supabase 클라이언트 노출 등 치명적인 보안 결함을 색출합니다.

### Step 3: SEO 및 메타데이터 점검 (Skill: `validate-og-meta`)
- 전역 `layout.tsx`의 메타데이터 완성도를 확인합니다.
- 각 라우팅 페이지 성격에 맞는 독립적인 OG 메타데이터(`title`, `description`, `images`) 설정 상태를 점검하고, 누락된 곳에 추천 코드를 도출합니다.

## 2. 통합 분석 리포트 생성 (Report Artifact)

3단계의 분석이 완료되면, 발견된 문제점과 수정 방향을 종합한 **Markdown Artifact (예: `feature_validation_report.md`)**를 생성하여 사용자에게 제공해야 합니다.

**리포트 필수 구성 요소:**
- **요약 (Summary)**: 각 분야별 Pass/Fail 상태 요약
- **1. 테스트 결과**: 프레임워크 유무, 커버리지, 발견된 결함
- **2. 설계 및 보안 이슈**: RSC/RCC 위반 사항, 보안 취약점 (위험도가 높은 경우 `> [!CAUTION]` 등의 GitHub 알럿 사용)
- **3. SEO 및 메타데이터**: 보완이 필요한 페이지 목록과 권장 설정 코드 스니펫
- **4. 시급성 및 액션 플랜**: 개발자가 당장 고쳐야 할 최우선 순위 작업 제안
